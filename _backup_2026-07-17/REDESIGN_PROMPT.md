# 健身紀錄 App 大翻新：筋肉人養成遊戲化改版

請先完整閱讀專案現有程式碼（index.html、js/app.js、css/style.css、sw.js、manifest.json），理解現有架構後規劃再動工。過程中請使用已安裝的技能：**Impeccable、UI UX PRO MAX、GSAP、Lenis**。

## 專案背景

這是一個 vanilla JS 的健身紀錄 PWA（無框架、無 build step），資料存在 localStorage（鍵 `fitnessApp_v1`）。現有功能：五種運動類型（重訓/室內外跑步/游泳/單車）、七部位動作庫與示範圖、組間計時器、PR 追蹤、SVG 圖表、訓練日曆、資料匯出入，以及一個初版的虛擬人物系統（XP 由全部紀錄純函數重算、六階段成長、程式生成 SVG 角色）。

本次任務是**整體視覺大翻新 + 角色系統深化**。所有現有工具功能必須保留，用戶既有的 localStorage 資料必須完全相容（不能要求用戶清資料重來）。

## 核心概念：角色是鏡子，不是寵物

角色不再只靠 XP 長大，而是**誠實反映用戶各部位的訓練數據**。用戶臥推 100kg 但深蹲只有 60kg，角色就會呈現上半身壯碩、下半身纖細的誇張漫畫身材。這個「失衡會被畫出來」的機制是整個 App 的靈魂——用幽默戳中用戶，驅動他去補弱項。

## 一、部位強壯度系統（本次核心）

### 1.1 部位分數（0–100）

沿用現有七部位分類（chest / back / legs / shoulders / biceps / triceps / core），每個部位計算一個 0–100 的強壯度分數：

1. 對該部位的每筆訓練組數計算預估最大肌力 e1RM = 重量 × (1 + 次數/30)（Epley 公式），取該部位歷史最佳。
2. **依體重相對化**：相對強度 = e1RM ÷ 用戶體重。
3. 用力量標準門檻把相對強度映射成 0–100 分（分段線性插值）。各部位錨點參考（相對強度 → 分數）：
   - 腿部（深蹲類）：0.5→20、1.0→50、1.5→75、2.0→95
   - 胸部（臥推類）：0.35→20、0.75→50、1.1→75、1.5→95
   - 背部（硬舉/划船/引體類）：0.5→20、1.0→50、1.5→75、2.0→95
   - 肩部（肩推類）：0.25→20、0.5→50、0.75→75、1.0→95
   - 二頭/三頭（彎舉/下壓類）：0.15→20、0.35→50、0.55→75、0.75→95
   - 核心：以訓練頻率與負重綜合計，錨點自行設計
   - 徒手動作（伏地挺身、引體向上等無重量紀錄）以次數換算等效強度，公式自行設計並註明
4. 動作名稱 → 部位的歸屬直接用現有 `PRESET_EXERCISES` 的分類；自訂動作歸入其所屬部位。

### 1.2 衰退機制（溫和 + 保底）

- 某部位超過 **7 天**未訓練後開始衰退：每天 -1 分。
- **保底**：不會低於歷史最高分的 60%。
- 衰退前 2 天，在首頁角色對應部位顯示提示（如腿部微微變暗 + 「該練腿了」氣泡），把損失趨避轉成行動提示。
- 分數計算必須是**純函數**：由全部訓練紀錄 + 今天日期推導，不存累計值，編輯/刪除紀錄自動修正。

### 1.3 有氧的歸宿

跑步/游泳/單車不長肌肉，餵「**體能與線條**」：
- 耐力分數（0–100）：由近 30 天有氧總距離/時長計算（也依體重相對化精神設計）。
- 視覺回饋：耐力高 → 角色線條更精實（腹肌線條浮現、輪廓收緊）+ 氣場特效（腳下火焰/風動線條，GSAP 動畫）。

## 二、角色視覺：筋肉人漫畫風

- 參考漫畫《筋肉人》（Kinnikuman）：**粗黑描邊、誇張肌肉塊、漫畫排線陰影、熱血誇張比例**。
- 程式生成 SVG，角色由部位參數驅動：肩寬、上臂圍、前臂、胸厚、腰身、大腿圍、小腿圍等各自對應部位分數縮放。**失衡要畫得明顯而好笑**（上壯下細就是經典雞腿身材）。
- 保留現有的身高/體重影響（BMI → 身寬、身高 → 腿長）與性別髮型。
- 等級/階段系統保留（六階段稱號與配件解鎖），但視覺改為筋肉人風格重繪。
- 訓練儲存成功的瞬間：剛練的部位在角色身上**發光 + 膨脹一下**（GSAP），這是全 App 最重要的一個回饋動畫。

## 三、視覺設計系統

- **主調：筋肉人熱血感**——高飽和紅/橘/黃、漫畫式集中線、爆裂感字體層次。
- **輔調：黑底金屬感**——按鈕、HUD、卡片邊框用槍鐵灰/鉻金屬漸層，NTC 式的深色質感。主從關係：熱血色是情緒與強調，黑金屬是介面骨架。
- 版式參考 Nike Training Club：超大粗體標題（中文字重要夠粗）、深色背景、全幅視覺區塊、內容卡片化、大量留白換成大量張力。
- 深色為預設主題。行動優先，維持 max-width 480px 的 App 容器。
- 動畫：GSAP 負責角色待機呼吸、升級爆發、部位發光、數字滾動；Lenis 負責頁面平滑滾動。過場要快（<300ms），不能拖慢紀錄操作。

## 四、頁面架構

- **首頁 = 角色舞台**：筋肉人站在畫面中央（GSAP 待機動畫），上方 HUD（等級、XP 條、連續天數），部位衰退提示直接標在角色身上，下方一顆大按鈕「開始訓練」。
- **訓練流程**：現有的選類型 → 選部位 → 記組數流程完整保留，換上新視覺。組間計時器保留。
- **角色頁**：部位分數雷達圖或部位條列（含各部位歷史最高與目前值）、耐力分數、階段路線圖、身體資料編輯。
- **戰績頁**：整合現有的日曆、PR、圖表、進度，用 NTC 式數據卡呈現。
- **onboarding**：新用戶建角色流程保留（暱稱/性別/身高/體重），換新視覺。

## 五、技術約束

- 維持 vanilla JS + 無 build step 的 PWA 架構。GSAP 與 Lenis 從 CDN（cdnjs）載入，並處理離線 fallback（動畫庫載入失敗時 App 功能不受影響）。
- localStorage 資料結構向後相容；如需擴充欄位，寫遷移邏輯。
- sw.js 快取版本號記得升級，CDN 資源加入快取清單。
- XP 與部位分數全部維持「由紀錄純函數重算」的設計原則。
- 完成後自我驗證：新用戶 onboarding 流程、舊資料載入不壞、記一筆臥推後胸部分數與角色視覺變化、衰退模擬（改系統日期或注入舊日期紀錄）、離線開啟。
